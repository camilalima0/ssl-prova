Neste passo, utilizamos o próprio OpenSSL para assinar o certificado (aluno.crt) usando a nossa própria chave privada (aluno.key), ignorando a necessidade de uma Autoridade Certificadora externa. Isso é chamado de "Certificado Auto-assinado" (Self-Signed). Ele fornece criptografia total para o tráfego de dados, o que é excelente para homologar APIs em Python ou Java em ambientes de desenvolvimento local. No entanto, como não há uma entidade terceira de confiança atestando a identidade do servidor, navegadores web emitirão alertas de segurança para os usuários finais.
